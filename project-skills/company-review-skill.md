# Skill: Company Review Article

## Purpose

Use this skill whenever the user asks to create, rewrite, audit, or update a review page for a Prop Firm, broker, crypto exchange, or funded trading company.

The goal is to produce a consistent, SEO-ready, comparison-ready article using the same structure every time.

## Required Inputs

If available, collect:

- Company name
- Official website URL
- Program types: 1-step, 2-step, instant funding, futures, crypto, broker
- Account sizes
- Prices
- Profit targets
- Daily drawdown
- Max drawdown
- Drawdown type
- Minimum trading days
- News trading rule
- Weekend holding rule
- EA/bot rule
- Copy trading rule
- Hedging rule
- Payout split
- Payout frequency
- Refund policy
- Scaling plan
- Restricted countries
- Platform: MT4, MT5, cTrader, DXtrade, Match-Trader, TradingView, etc.
- Trust notes: regulation, legal entity, warnings, user complaints, official status

If data is missing, mark it as `غير مذكور بوضوح` instead of inventing.

## Article Structure

Use this exact structure:

1. H1: مراجعة [Company Name]
2. Intro paragraph
3. Quick Verdict Box
4. Company Snapshot Table
5. Account Plans Table
6. Rules Table
7. Payouts and Withdrawals
8. Trading Conditions
9. Allowed and Restricted Strategies
10. Pros and Cons
11. Who Is This Company Best For?
12. Who Should Avoid It?
13. Pricing Analysis
14. Trust and Transparency Notes
15. Best Alternatives
16. Final Verdict
17. FAQ
18. Official Source Links

## Quick Verdict Box

Must include:

- Best for
- Not suitable for
- Main strength
- Main weakness
- Risk level: Low / Medium / High
- Review status: Verified / Needs Update / Partial Data

## Writing Rules

- Arabic language.
- Clear, direct, non-promotional.
- Do not exaggerate.
- Do not claim a company is safe unless there is strong evidence.
- Mention risks clearly.
- No invented numbers.
- If data is not confirmed, write `غير مؤكد`.
- Prefer official sources.
- Avoid duplicated generic paragraphs.
- Every company must have a unique verdict.

## SEO Requirements

Title format:

`مراجعة [Company Name] [Year]: القواعد، الأسعار، السحب، وهل تستحق؟`

Meta description format:

`مراجعة شاملة لشركة [Company Name]: الخطط، الأسعار، قواعد السحب، التداول وقت الأخبار، المميزات والعيوب، والحكم النهائي قبل شراء التحدي.`

Suggested slug:

`reviews/company-name.html`

## FAQ Questions

Include 5 to 7 questions:

1. هل [Company Name] موثوقة؟
2. ما أقل سعر تحدي في [Company Name]؟
3. هل تسمح [Company Name] بالتداول وقت الأخبار؟
4. هل تسمح [Company Name] بالاحتفاظ بالصفقات آخر الأسبوع؟
5. متى يتم السحب في [Company Name]؟
6. هل [Company Name] مناسبة للمبتدئين؟
7. ما أفضل بديل لـ [Company Name]؟

## Final Output

When producing a review, output:

1. Clean article content.
2. Tables in consistent format.
3. SEO title.
4. Meta description.
5. Suggested internal links.
6. Missing data list.
7. Data confidence score from 1 to 10.
